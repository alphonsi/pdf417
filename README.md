# PDF417 Barcode Implementation

This project provides a complete PDF417 barcode generation and decoding solution using Python.

## Overview

PDF417 is a two-dimensional stacked bar code symbology capable of encoding large amounts of data. This implementation provides:

- **Encoding**: Generate PDF417 barcodes from text data using the `treepoem` library
- **Decoding**: Read PDF417 barcodes from images using the `pyzbar` library
- **Testing**: Comprehensive test suite to verify encoding/decoding functionality

## Requirements

Install the required dependencies:

```bash
pip install treepoem pyzbar pillow
```

Note: You may also need to install ZBar for pyzbar to work properly on some systems.

## Usage

### Basic Encoding

```python
import treepoem
from PIL import Image

# Generate a PDF417 barcode
data = "Hello World!"
barcode_image = treepoem.generate_barcode(
    barcode_type='pdf417',
    data=data,
    options={
        "columns": 12,
        "security_level": 4,
        "rows": 0
    }
)

# Convert to black/white and save
barcode_image = barcode_image.convert('1')
barcode_image.save("barcode.png")
```

### Basic Decoding

```python
from pdf417decoder import PDF417Decoder
from PIL import Image

# Load and decode an image
img = Image.open('barcode.png')
decoder = PDF417Decoder(img)
result = decoder.decode()
print(f"Decoded data: {result}")
```

### Advanced Decoding

```python
# Decode all barcodes in an image
all_results = decoder.decode_all()

# Get detailed barcode information
if hasattr(decoder, '_barcodes_info'):
    for i, info in enumerate(decoder._barcodes_info):
        print(f"Barcode {i+1}: {info['data']}")
        print(f"Position: {info['rect']}")
```

## Test Results

### Current Status

- ✅ **Encoding**: PDF417 barcode generation works correctly
- ❌ **Decoding**: pyzbar library cannot detect barcodes generated by treepoem
- ⚠️  **Alternative**: Consider using different library combinations or manual decoding

### Generated Barcodes

The system successfully generates PDF417 barcodes with:
- Proper dimensions (e.g., 546x30 pixels)
- Correct black/white format
- Standard PDF417 structure

### Decoding Issues

The current implementation uses `pyzbar` for decoding, but it cannot detect PDF417 barcodes generated by `treepoem`. This is a known compatibility issue between some barcode libraries.
